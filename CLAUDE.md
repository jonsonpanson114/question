# 問いの道場 - Claude Code プロジェクト設定

## プロジェクト概要

**名称**: 問いの道場 (Toi no Dojo)
**説明**: フォローアップクエスチョンを磨く練習アプリ
**技術スタック**: Next.js 16, React 19, Gemini AI, PWA

---

## 🎯 プロジェクトの目的

1. **メイン機能**: 4つの練習モードで質問スキルを向上させる
2. **既存技術**: Gemini APIによるAIフィードバック
3. **PWA対応**: オフライン動作、インストール可能

---

## 🚫 絶対禁止事項

### セキュリティ
1. **秘密情報をドキュメント化しない**
   - APIキー、秘密鍵はドキュメントに書かない
   - `.env.example` に変数名のみ記載
   - `.gitignore` で `.env.local` を除外済み確認

2. **GitHubに秘密情報をプッシュしない**
   - コミット前に `git diff` で秘密情報の有無を確認
   - `git status` で意図しないファイルが含まれていないか確認

### 既存機能保護
3. **既存APIへの変更を最小限にする**
   - `app/api/dojo-*` ファイルは原則変更しない
   - 新機能は完全に独立したパスで実装する
   - 既存UIへの変更は最小限にする

4. **既存コードの破壊を防ぐ**
   - 大きな変更前には必ず計画を立てる
   - ユーザーの承認を得てから実装する
   - 各ステップで既存機能を確認

## 📋 開発プロセス

### 1. 計画立案フェーズ
- 目的と範囲を明確に定義

- セキュリティリスク評価

### 2. 実装フェーズ
- 小さな単位で実装

- 問題があれば即座に報告

### 3. テストフェーズ
- 既存機能の回帰テスト
- 新機能のテスト
- セキュリティチェック

## 📊 既存機能一覧

### 練習モード
1. **日常雑談** (casual) - 日々の会話を深める問い
2. **仕事の1on1** (oneonone) - 問いで場を耕す
3. **自己内省** (introspect) - 自分の内側を探る
4. **読書振り返り** (book) - 言葉を問いに変える

### APIエンドポイント
- `/api/dojo-feedback` - フィードバック評価
- `/api/dojo-message` - 開始メッセージ
- `/api/dojo-reply` - ラリー返信
- `/api/dojo-scenario` - シナリオ生成

## ⚡ Claude Code活用ガイド

### 計画時
1. まず CLOUD.md に詳細計画を記述
2. ユーザーに計画を提示
3. 承認を得てから実装

### 実行時
1. 各ステップで既存機能への影響を確認
2. 問題があれば即座に報告
3. 小さな単位で実装

### セキュリティ
1. 秘密情報は変数名のみ記載
2. 実データは環境変数でのみ使用
3. コミット前に `git diff` で確認
